Description

Usb LIN(Local interconnect network) Converter(ULC) works as LIN to USB converter. Two communications port exist, USB(virtual COM port) and side USART port. UCCB can work both with PC or Your custom embedded device. Multiple features are supported like:

  • LIN 2.1 support
  • Listen only mode, for all frame reception, and sending single frames.
  • Master mode, with up to 15 frame slots
  • 19200 baud rate
  • External 7-12V is needed for LIN network
  • Device size is 31x18 mm (without USB connector)
  • Termination resistors controlled by software
  • USB dfu compatible bootlader for new features
  • SLCAN compatible, works with LINUX socket-can and some of CAN-utils

    • After USB enumeration device creates virtual COM port that is used both for data transfer and converter setup. Frames are send on USART only if no HOST is connected to USB. Device is be powered directly from USB, External 12V is needed to supply LIN transcriver. Board layout is shown below.

    COM port Commands

    Below is list of all available commands. All those commands can be send by USB or P4 port USART. USART baud rate is virtual so can be any. For the newest list see SLCAN.c file in source codes. All commands are ASCI based. Commands are line based and terminated with newline character CR (0xD). On error the device responses with 0x7. To start LIN frame reception just type "L" as Listen mode command.
    Command Description
    O Opens device in master mode.
    C Close LIN port on device, frame will not be received.
    Sx Currently no baud rate selection
    V Get hardware version.
    v Get firmware version.
    N Get serial number.
    L Open device in, silent, message analyzer mode.
    tiiildd In Silent mode, transmits full header and data frame.
    In Master mode, puts or updates table set with new frame
    0ii: Identifier in hexadecimal format (000-0FF)
    l: Data length (0-8)
    dd: Data byte value in hexadecimal format (00-FF)
    riiil In Silent mode, transmits header only with set ID and Lenght, subscibed slave should answer the header.
    In Master mode, puts or updates table set with this id for continous sending. Proper timing is set according to LIN standard.
    0ii: Identifier in hexadecimal format (000-0FF)
    l: Data length (0-8)
